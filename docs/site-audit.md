# 站点系统审计：用户视角分析

## 模拟用户场景

一个玩家刚买了 Subnautica 2，遇到问题，搜索「深海迷航2 银矿在哪」来到我们站。

### 用户旅程分析

1. **着陆** → 搜索引擎点进某篇攻略
2. **扫一眼** → 3 秒内判断「这个站靠谱吗」
3. **找答案** → 快速定位到需要的信息
4. **可能探索** → 看完后点击相关内容
5. **可能收藏** → 觉得有用就加书签

### 每个环节的问题

---

## 一、已实现且有效的功能

| 功能 | 评价 | 用户价值 |
|------|------|----------|
| 版本标签 | ✅ 好 | 玩家知道内容是否过时 |
| 恐惧症标签 | ✅ 差异化 | 独特卖点 |
| 配方计算器 | ✅ 实用 | 工具留存 |
| 互动地图框架 | ⚠️ 有框架但无真实数据 | 目前无用 |
| Pagefind 搜索 | ✅ 好 | 站内搜索刚需 |
| 移动端菜单 | ✅ 有 | 基础功能 |
| FAQ + Schema | ✅ SEO 好 | 搜索引擎友好 |

## 二、鸡肋/无用的功能（应该砍掉或重做）

| 功能 | 问题 | 建议 |
|------|------|------|
| **科技树可视化** | 数据太少（10 个物品），图几乎是空的 | 暂时隐藏，等数据齐了再上 |
| **互动地图** | 没有真实瓦片，占位 SVG 无意义 | 保留框架但首页不要主推 |
| **深度分布图** | 只有 4 个群落，条形图太空 | 等群落数据齐了再显示 |
| **版本日志页** | 只有 2 条，看起来很荒凉 | 合并到首页一个小区块 |
| **Biomod 页** | 几乎没有数据，只有框架文字 | 合并到攻略里，不单独成页 |
| **载具页** | 只有占位表格 | 合并到物品页的载具分类 |
| **贡献指南页** | 目前没有社区，无人贡献 | 放 footer 链接就行 |

## 三、缺失但用户急需的功能

| 缺失 | 用户场景 | 优先级 |
|------|----------|--------|
| **快速答案卡片** | 用户搜「银矿在哪」想要一句话答案 | P0 |
| **步骤高亮框** | 攻略里的关键步骤应该视觉突出 | P0 |
| **警告/提示框** | ⚠️ 危险提示、💡 小贴士 | P0 |
| **坐标复制按钮** | 玩家想复制坐标到游戏里 | P1 |
| **阅读进度条** | 长文顶部显示阅读进度 | P2 |
| **返回顶部按钮** | 长页面必备 | P1 |
| **上一篇/下一篇** | 流程攻略需要线性导航 | P1 |

## 四、视觉问题（从截图分析）

| 问题 | 原因 | 修复 |
|------|------|------|
| 正文太散 | 列表间距过大 | ✅ 已修 |
| 没有视觉重点 | 全是同样大小的文字 | 需要加提示框/高亮框 |
| 首页太空 | 内容少但布局占位大 | 重新设计首页 |
| 导航太多项 | 12 个导航项太拥挤 | 精简到 7-8 个核心 |
| 页脚太重 | 三栏页脚对小站来说太正式 | 简化为单行 |

## 五、立即执行的优化计划

### Round 1：内容展示优化（最影响用户体验）
1. 加 Callout 组件（提示/警告/步骤框）
2. 精简导航（砍掉鸡肋页面的入口）
3. 首页重写（面向「刚买游戏的玩家」）
4. 加返回顶部按钮

### Round 2：砍掉鸡肋
5. 隐藏科技树（导航移除）
6. 隐藏载具独立页（合并到物品）
7. 隐藏 Biomod 独立页（合并到攻略）
8. 版本日志合并到首页

### Round 3：增强核心页面
9. 攻略页加上一篇/下一篇导航
10. 攻略页加阅读进度条
11. 首页加「热门问题」快速入口
